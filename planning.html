üìù Task Anatomy

Every ClickUp task should have these fields filled:

Title
Action verb + specific outcome (e.g., "Implement user login API endpoint")
Description
Context, requirements, and any relevant links
Acceptance
Gherkin-style Given/When/Then scenarios
Size
T-shirt size: XS, S, M, L (no XL‚Äîbreak it down)
Branch
Auto-generated or manual: feature/TASK-123-short-description

XS 15-30 min ‚Ä¢ Config change, typo fix
S 30-60 min ‚Ä¢ Single function, simple endpoint
M 2-4 hours ‚Ä¢ Feature with tests
L 4-8 hours ‚Ä¢ Complex feature, multiple files

If you're sizing something as L, ask: "Can I split this into two M tasks?" Usually yes.

ü•í Gherkin Acceptance Criteria

Write acceptance criteria in Gherkin format. This is the contract between you and the AI.

üìã Example: Login Feature

Feature: User Authentication

Scenario: Successful login with valid credentials
  Given a registered user with email "test@example.com"
  And password "SecurePass123!"
  When they submit the login form
  Then they receive a JWT token
  And are redirected to the dashboard

Scenario: Failed login with invalid password
  Given a registered user with email "test@example.com"
  When they submit login with wrong password
  Then they see error "Invalid credentials"
  And no token is issued

Scenario: Rate limiting after failed attempts
  Given 5 failed login attempts from same IP
  When a 6th attempt is made
  Then return 429 Too Many Requests
  And include retry-after header

üìÑ todo.md Template

# Task: TASK-123 - Implement User Login API

## Objective
Create POST /api/auth/login endpoint with JWT response

## Acceptance Criteria
- [ ] Validates email format
- [ ] Checks password against bcrypt hash
- [ ] Returns JWT with 24h expiry
- [ ] Returns 401 for invalid credentials
- [ ] Rate limits to 5 attempts per minute

## Implementation Plan
1. [ ] Create LoginRequest schema
2. [ ] Create LoginResponse schema
3. [ ] Implement authenticate_user() service
4. [ ] Create /login endpoint
5. [ ] Add rate limiting middleware
6. [ ] Write unit tests
7. [ ] Write integration tests

## Files to Modify
- src/schemas/auth.py
- src/services/auth_service.py
- src/api/routes/auth.py
- tests/test_auth.py

## Notes
- Follow pattern in @src/api/routes/users.py
- Use existing JWT utils in @src/utils/jwt.py
